# ТЗ: Обработчик квеста чернокнижника

## Статус
> Готово к реализации

---

## Описание задачи

Скрипт автоматически принимает квест у чернокнижника, проходит лабиринт до выхода
и забирает награду. Лабиринт проходится методом DFS с возвратом из тупиков.
Состояние (карта, позиция, история ходов) сохраняется в `chrome.storage`, потому что
каждый шаг — это перезагрузка страницы.

---

## Страницы и типы

Все страницы квеста — это `quest.php`, заголовок `.contur strong` = **"Чернокнижник"**.
Тип страницы определяется последовательной проверкой:

```
1. Есть ссылка "Завершить маршрут"?         → finish (выход найден)
2. Есть "справился с заданием" в .news_full? → earn (выбор награды)
3. Есть "Куда пойти:" в .news_full?          → process (шаг лабиринта)
4. Иначе                                     → ask (диалог, принять квест)
```

### Тип 1: ask — принятие квеста
**Файл:** `quest-warlock-ask.html`

`.news_full` содержит диалог чернокнижника (без сетки).
В `.error .header_mes` два варианта ответа.

**Действие:** кликнуть на ссылку с текстом `Берусь за это`, инициализировать состояние в storage.

```js
const links = [...document.querySelectorAll('.error .header_mes a')];
const acceptLink = links.find(a => a.textContent.includes('Берусь за это'));
// acceptLink.href → quest.php?sel=2547885&...
```

**При клике — записать в storage:**
```js
wor_quest_warlock_active   = true
wor_quest_warlock_pos      = { x: 0, y: 0 }
wor_quest_warlock_visited  = ["0,0"]
wor_quest_warlock_history  = []
wor_quest_warlock_steps    = 0
```

### Тип 2: process — шаг лабиринта
**Файлы:** `quest-warlock-process.html`, `quest-warlock-process2.html`

`.news_full` содержит текст `Куда пойти:` и 9 `<img>` тегов (сетка 3×3).

**Структура сетки — индексы img:**
```
[0] [1] [2]   ← верхний ряд
[3] [4] [5]   ← средний (игрок всегда [4])
[6] [7] [8]   ← нижний ряд
```

**Значения src:**
| src | Значение |
|-----|----------|
| `images/lab_stena.gif` | Стена — нельзя идти |
| `images/lab_igrok.gif` | Игрок (всегда [4]) |
| `images/lab_pusto.gif` | Пусто — можно идти |
| `images/lab_luk.gif`   | **Выход** — триггер финиша |

**Маппинг индексов → направление → дельта координат:**
| img индекс | Направление | Дельта (dx, dy) | Обратное |
|-----------|-------------|-----------------|---------|
| [1] | `up`    | (0, -1) | `down` |
| [7] | `down`  | (0, +1) | `up` |
| [3] | `left`  | (-1, 0) | `right` |
| [5] | `right` | (+1, 0) | `left` |

**Парсинг сетки:**
```js
const imgs = [...document.querySelectorAll('.news_full img')];
const dirMap = { up: imgs[1], down: imgs[7], left: imgs[3], right: imgs[5] };

function getImgType(img) {
    if (img.src.includes('lab_stena')) return 'wall';
    if (img.src.includes('lab_pusto')) return 'empty';
    if (img.src.includes('lab_luk'))   return 'exit';
    return 'unknown';
}
```

**Парсинг доступных ходов (из ссылок, а не из картинок):**
```js
const moveLinks = [...document.querySelectorAll('.error .header_mes a')];
// Тексты: "Идти вверх" / "Идти вниз" / "Идти влево" / "Идти вправо"
const dirFromText = { 'вверх': 'up', 'вниз': 'down', 'влево': 'left', 'вправо': 'right' };

const available = {}; // { up: href, down: href, ... }
moveLinks.forEach(a => {
    const key = Object.keys(dirFromText).find(k => a.textContent.includes(k));
    if (key) available[dirFromText[key]] = a.href;
});
```

**Действие:** выполнить алгоритм DFS (см. ниже), перейти по href выбранного направления.

## Алгоритм DFS (применяется на странице типа process)

### Структуры данных в chrome.storage

```js
wor_quest_warlock_active   // bool — идёт ли квест сейчас
wor_quest_warlock_pos      // { x, y } — текущая позиция
wor_quest_warlock_visited  // string[] — посещённые клетки ["0,0", "1,0", ...]
wor_quest_warlock_history  // string[] — стек ходов ["down", "right", ...] для бэктрекинга
wor_quest_warlock_steps    // number — счётчик шагов (для уведомления)
```

### Порядок предпочтения направлений (при нескольких свободных)

```
right → down → left → up
```

Смещение в сторону правого нижнего угла повышает вероятность найти выход быстрее,
но алгоритм не зависит от этого — он всё равно обойдёт весь лабиринт если нужно.

### Псевдокод шага

```
1. Загрузить состояние из storage: pos, visited, history

2. Отметить pos как посещённую: visited.add(`${pos.x},${pos.y}`)

3. Получить доступные направления: available = { up: href, ... }

4. Приоритетный список: ['right', 'down', 'left', 'up']

5. Найти первое направление dir из списка, где:
   - dir есть в available
   - клетка (pos.x+dx, pos.y+dy) НЕ в visited

6. Если такой dir найден:
   - history.push(dir)
   - pos = { x: pos.x+dx, y: pos.y+dy }
   - Сохранить состояние в storage
   - window.location.href = available[dir]

7. Если dir не найден (тупик):
   - backDir = opposite[history.pop()]
   - pos = { x: pos.x + deltas[backDir].dx, y: pos.y + deltas[backDir].dy }
   - Сохранить состояние в storage
   - window.location.href = available[backDir]
   // backDir гарантированно есть в available — мы пришли оттуда
```

## Кнопка остановки

Аналогично другим модулям (алхимия, добыча и т.д.) — на страницах квеста отображается
фиксированная кнопка управления через `CommonHelper.createDisableButton()`.

Кнопка показывается если `wor_quest_warlock_opt_active_button = true` (настраивается в popup).

**Когда квест активен** (`wor_quest_warlock_active = true`):
```js
CommonHelper.createDisableButton('Остановить квест', async () => {
    await CommonHelper.setExtStorage('wor_quest_warlock_active', false);
    await CommonHelper.setExtStorage('wor_quest_warlock_pos', null);
    await CommonHelper.setExtStorage('wor_quest_warlock_visited', []);
    await CommonHelper.setExtStorage('wor_quest_warlock_history', []);
    await CommonHelper.reloadPage();
});
```

Кнопка отображается на страницах типа **process** и **finish** — т.е. во время активного
прохождения лабиринта. На странице **ask** кнопка не нужна (там инжектированная ссылка запуска).

| Ключ | Тип | Описание |

| `wor_quest_warlock_active` | boolean | Идёт ли прохождение квеста |

| `wor_quest_warlock_history` | string[] | Стек ходов для бэктрекинга |

## Механизм запуска

Квест запускается **только вручную** — никакого авто-старта по таймеру или из popup.

Реализация: скрипт на странице **ask** добавляет в блок `.error .header_mes` дополнительную
ссылку (инжектированную в DOM, не серверную):

```
» Включить автовыполнение квеста       ← добавляем сами
» У меня сейчас нету на это времени…  ← серверные
» Хорошо. Берусь за это.
```

При клике на "Включить автовыполнение квеста":
1. Записать `wor_quest_warlock_active = true` в storage
2. Автоматически кликнуть на серверную ссылку "Хорошо. Берусь за это." (с задержкой)

Если игрок нажимает серверные ссылки напрямую — скрипт не вмешивается.
Автовыполнение работает только если `wor_quest_warlock_active = true`.

**Что НЕ входит в эту версию:**
- Запуск из popup
- Автонавигация до чернокнижника из произвольной точки карты
